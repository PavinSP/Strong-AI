# Slide 31-36: Brain Architecture (The Block Diagram)

- Header: **3. Brain Architecture**
- Subtitle: **The Thought Machine**
- Core Visuals:
  - **Slide 32**: A massive 3D cube representing the **L Brain**. Inside the cube are four modular blocks:
    1. **I Mind** (Central Logic Unit / CPU)
    2. **H Memory** (Persistence / Hard Drive)
    3. **J Language Center** (I/O Protocol)
    4. **K Motion Center** (Kinematics / Actuators)
  - **Slide 34**: The Memory block is partitioned into specialized databases (Episode, Values, Motion, Language, Knowledge, Planning).
  - **Slide 35**: The Mind block is partitioned into specific execution threads (Understanding, Reasoning, Learning, Logic, Initiating).

### Conceptual Explanation

This is the holy grail of the seminar: the literal system architecture for an Artificial General Intelligence. 

It proves that an AGI is not one massive, black-box Neural Network. It is a highly modular, decoupled system. The `I Mind` (the logic engine) is completely separated from the `H Memory` (the database). The `J Language` and `K Motion` centers act as independent API drivers handling external communication and physical movement. 

### Why this matters

This solves the "Catastrophic Forgetting" and "Hallucination" problems of modern LLMs. Current LLMs try to smash Memory, Logic, and Language into a single homogenous matrix of weights. This architecture dictates that they must be cleanly separated into distinct microservices.

# Slide 37-43: Brain Screens and Thought Windows

- Header: **4. Brain Screens / 5. Thought Window**
- Core Concepts:
  - **Slide 38**: Sensors (Vision, Hearing) push data into the Brain. Actuators (Moving, Speaking) pull data out.
  - **Slide 42**: "Screens" are the internal interfaces (APIs) between the brain sections.
  - **Slide 43**: Thoughts are *Association Networks* displayed on a screen. They are not a sequential channel. The "Mind's Eye" is simply the part of the screen the brain is currently focused on. 
  - **Slide 40**: This internal visual protocol is called **GREBTL** (Generic Role-Entity-Based Thought Language).

### Conceptual Explanation

If the Brain is a collection of modular blocks, how do they talk to each other? They use "Screens" (Internal APIs). 
When the Vision sensor detects a spider (Slide 39), it renders an association network onto the visual "Screen." The `I Mind` looks at that screen, validates it against the `Value Memory` ("spiders are bad"), and immediately renders an Action Plan ("Run Away!") onto the Motion "Screen." The `K Motion` center reads that screen and fires the physical leg motors. 

### Why this matters

This explicitly defines the **Data Payload** passed between AGI microservices. The payload is not raw English text (sequential). The payload is a standardized Knowledge Graph (GREBTL).

# Slide 44-45: AI Spots (Machine Learning)

- Header: **6. AI Spots**
- Subtitle: **Using Machine Learning**
- Core Visual (Slide 45):
  - Shows the Brain block diagram, but with traditional "AI" (Machine Learning models) acting as isolated utility scripts:
    - **Visualiser/Editor AI**: Translates screen language to diagrams.
    - **Real Time Detector AI**: Translates raw sensor data into screen language.
    - **Translator AI**: Translates Natural Language to internal Screen Language.
    - **Value Improver AI**: Updates the Value database.

### Conceptual Explanation

This slide drops a massive paradigm shift: **Machine Learning (Deep Learning / LLMs) is NOT Strong AI.** 
Deep Learning models are just "Spots"—dumb, highly specialized utility scripts used as translators on the edges of the true AGI. You use a Neural Network to translate audio waves into text, or pixels into bounding boxes. But the actual "Intelligence" (the Brain) is the deterministic graph-logic engine sitting in the center.
