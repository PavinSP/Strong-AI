# Slide 5-8: Types of Language

- Header: **1. Types**
- Core Taxonomies:
  - **Natural Languages**: Human / Animal.
  - **Artificial Languages**: Formal languages (Programming, Modeling, Semantic Networks) and constructed human languages (Esperanto).
  - **Thought Languages**: Just mental (internal mental entities and associations).
  - **Expression (Output)**: Oral, Body Language, Sign Language, Writing.
  - **Impression (Input)**: Auditory, Visual, Tactile.
  - **Medium**: Sound waves, Photons, Persistent Materials (Paper).
- Visual (Slide 8): A massive red **STOP** sign.

### Conceptual Explanation

These slides categorize language purely as a data transmission protocol. The STOP sign on Slide 8 is a perfect example of an optical, non-verbal artificial language used for "Action Control." The sign transmits an overriding command via photons into the visual cortex of a driver to halt physical motion. 

### Why this matters

For AI, this proves that an LLM generating English text and a compiler generating C++ bytecode are doing the exact same thing. They are using an agreed-upon protocol (Syntax/Grammar) to serialize a mental graph so it can be transmitted across a medium to control another system.

# Slide 9-13: Purpose (Action Control)

- Header: **2. Purpose**
- Subtitle: **Action Control and Learning not by Doing**
- Core Concepts:
  - **Slide 10**: Language creates a feedback loop between `communication` (externalizing thoughts) and `thought structuring`. 
  - **Slide 11**: Natural Language is "Moving Materialised Information" from one human brain directly into another human brain.
  - **Slide 12**: Artificial/Formal Language is moving an "Action Plan (Programme)" from a human brain directly into a **CPU** to control a machine's Memory.
  - **Slide 13**: Language is a tool for controlling the action and behavior of socio-technical systems.

### Conceptual Explanation

This is a massive conceptual leap. The presenter explicitly defines writing computer code (Formal Language) as a form of communication. When a programmer writes Java, they are translating their internal mental graph into a formal language and transmitting it into a silicon CPU. 
Language also enables "Learning not by Doing." Instead of an agent having to touch fire to learn it is hot (Learning by Doing), another agent can just transmit a linguistic schema ("fire is hot") over soundwaves. 

### Why this matters

This is the exact mechanic behind **Prompt Engineering** and **Zero-Shot Learning**. You can instantly update the internal logic of an LLM by transmitting an English prompt. You bypass millions of epochs of gradient descent (Learning by Doing) by simply transmitting a direct linguistic update.

# Slide 14-17: Syntax (Structure)

- Header: **3. Syntax**
- Core Concepts:
  - Syntax defines the physical order of the data packets.
  - **Slide 16**: Natural Language syntax is usually `Subject` $\rightarrow$ `Predicate` $\rightarrow$ `Object`. ("The tomorrow of yesterday is the yesterday of tomorrow.")
  - **Slide 17**: Procedural Computer Language syntax uses the structure: `if (Condition) then Statement_True else Statement_False`.

### Conceptual Explanation

Syntax has nothing to do with meaning. It is strictly the structural blueprint of the packet. If the packets arrive out of order, the compiler crashes.

# Slide 18-20: Grammar (Rules)

- Header: **4. Grammar**
- Core Concepts:
  - **Slide 19 (Morphology)**: Adding an `s` to "yesterday" creates the plural "yesterdays."
  - **Slide 20 (EBNF)**: Shows the **Extended Backus-Naur Form**. `conditional statement ::= "if (" condition ") then" statement [ "else" statement ] ";"`

Grammar defines the mathematical rules of the language protocol. The presenter explicitly uses EBNF (the standard notation used to define computer programming languages) to explain grammar, proving that linguistic grammar and compiler grammar are identical logic systems.

# Slide 21-23: Semantics (Meaning)

- Header: **5. Semantics**
- Core Concepts:
  - Uses the test sentence: *"The tomorrow of yesterday is the yesterday of tomorrow."*
  - **Slide 22**: Maps the specific words to their semantic operations (`identity`, `amplification`, `belonging`).
  - **Slide 23**: Reduces the sentence to algebra. 
    - Day B: Yesterday = `T - 1`
    - Day A: Tomorrow of Yesterday = `(T - 1) + 1` = `T`
    - Meaning of the sentence: `T = T`. (A tautology). "Today is Today."
  - Question: *Does this sentence have any meaning at all?*

### Conceptual Explanation

Semantics is the final compiling step. It is the process of mapping the raw text tokens back into the actual mathematical Graph inside the brain. The mathematical reduction in Slide 23 proves that a sentence can be perfectly valid in Syntax and Grammar, but semantically collapse into a useless tautology. 

### Why this matters

This perfectly describes the problem of **LLM Hallucinations**. Language models are brilliant at Syntax and Grammar (they construct perfect-sounding sentences). But if they lack a grounded internal Knowledge Graph (Semantics), they will output mathematically hollow tautologies or logical contradictions.
